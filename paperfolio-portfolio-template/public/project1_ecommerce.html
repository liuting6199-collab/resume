Portfolio Project 01

电商用户转化漏斗分析与流失定位

基于1,500条用户行为数据，拆解拼多多场景下"浏览→点击→加购→下单→支付"全链路转化，定位最大流失环节并提出优化策略

Python Pandas 漏斗分析 渠道归因 用户分层 Matplotlib
项目背景与目标

某电商平台1月份产生1,500条用户行为记录，覆盖浏览、点击、加购、下单、支付全链路。发现整体转化率低于行业均值，希望定位流失最严重的环节，针对性优化。

核心问题：

  • 哪个环节流失用户最多？流失率是多少？
  • 不同渠道（搜索/推荐/百亿补贴/活动页/分享）的转化差异？
  • 不同品类、设备、用户类型的转化表现如何？
  • 如何量化优化后的潜在收益？

数据说明与清洗

数据集包含1,500条用户行为记录，覆盖2024年1月1日-1月31日。每条记录包含用户ID、设备类型、来源渠道、商品品类、用户类型、行为链路（浏览/点击/加购/下单/支付）及订单金额。

数据字段

user_id（用户ID）· device（设备）· channel（渠道）
category（品类）· user_type（用户类型）· view/click/add_cart/order/pay（行为）
amount（订单金额）· date（日期）

清洗过程

① 去重：无重复记录
② 缺失值：无缺失字段
③ 异常值：订单金额在合理范围内
④ 逻辑校验：行为链路符合顺序（支付必经过下单）

分析思路与假设

从原始行为日志出发，按"漏斗概览 → 流失定位 → 渠道拆解 → 品类对比 → 用户分层 → 设备分析"逐层下钻。

漏斗概览 → 流失定位 → 渠道拆解 → 品类对比 → 用户分层 → 设备分析

核心假设：① 百亿补贴渠道"高点击、低加购"；② 活动页渠道转化效率最低；③ 会员用户转化率显著高于新用户；④ 移动端转化率低于PC端。

数据可视化与分析
核心发现
  • 点击→加购是最大流失环节（56.9%）：1,028个点击用户中仅443个加购，超过一半用户点击后流失。说明商品详情页吸引力不足，或价格/评价未达预期。这是优化ROI最高的环节。
  • 分享渠道"少而精"，活动页完全无效：分享渠道整体转化率23.6%为全渠道最高，而活动页仅3.2%。活动页获券用户缺乏即时购买动机，建议暂停活动页发券，将预算转移至分享和搜索渠道。
  • 百亿补贴"高点击、低加购"：点击率85%全渠道最高，但加购率仅35%，用户被低价吸引但决策犹豫。需在商品页增加紧迫感元素（限时、库存紧张）促使用户立即加购。
  • 数码品类客单价最高但转化偏低：数码客单价¥329为全品类最高，但整体转化率仅12.6%（低于均值14.7%）。高客单用户决策周期长，需提供更多信任背书（评价、对比、保障）。
  • 会员转化率16.2% vs 新用户11.8%：会员用户转化率比新用户高37%，且客单价更高。建议加大会员权益投入，同时针对新用户设计"首单优惠"降低决策门槛。
  • PC端转化率（18.5%）显著高于移动端（13.2%）：PC用户购买意图更强，但流量占比仅15%。移动端体验优化是提升整体转化的关键。

业务建议与预期收益
优化商品详情页（ROI最高） 针对点击→加购流失56.9%，优化详情页首屏信息、增加用户评价、明确价格优势。若加购率提升至55%，预计每月增加订单180+，增收¥20,000+。
暂停活动页发券 活动页转化率仅3.2%，将预算转移至分享和搜索渠道。若将活动页50%预算转移至分享渠道，预计整体转化率提升至16.5%。
百亿补贴加紧迫感 在百亿补贴商品页增加"仅剩XX件""XX人正在看"等提示，利用损失厌恶心理。若加购率从35%提升至45%，预计每月增加订单120+。
扩大分享裂变机制 分享渠道转化效率最高（23.6%），建议增加分享奖励（双方各得优惠券），扩大高质量获客渠道。预计分享流量占比可从14%提升至20%。
会员权益升级 会员转化率比新用户高37%，建议推出会员专属价、免邮券、生日礼等权益，提升会员占比从25%至35%。
移动端体验优化 PC端转化率（18.5%）比移动端（13.2%）高40%，需优化移动端详情页加载速度、简化下单流程、增加一键支付。
分析局限性与后续方向
  • 数据周期：仅覆盖1个月数据，季节性因素（如春节）可能影响用户行为，建议补充季度数据验证结论稳定性。
  • 用户画像：缺少用户年龄、地域、消费频次等维度，无法做更精细的用户分层（如RFM模型）。
  • 因果推断：当前分析为相关性分析，建议后续通过A/B测试验证优化建议的实际效果。
  • 竞品对比：缺少行业基准数据，无法判断当前转化率在行业中的相对位置。
