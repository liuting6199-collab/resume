## 🎯 项目简介

OfferGet 是一个基于 AI 的求职辅助工具，帮助求职者系统化地准备求职全流程。通过结构化的方法论和 AI 辅助，将零散的工作经历转化为有竞争力的求职材料。

**核心解决的问题：**
- ❌ 工作经历零散，不知道如何组织
- ❌ 简历描述过于学生气，缺乏专业感
- ❌ 面试准备没有针对性，临场发挥
- ❌ 对目标公司和岗位了解不足

**OfferGet 的解决方案：**
- ✅ 场景化经历深挖：基于实际材料，延伸想象真实工作场景
- ✅ 量化成果表达：用具体数字体现工作价值
- ✅ 针对性面试准备：基于 JD 和经历生成高频问题预案
- ✅ 全流程闭环管理：从经历整理到面试通过，一站式解决

## 💡 核心设计哲学

| 原则 | 说明 |
|------|------|
| **解决问题导向** | 不是"我做了什么"，而是"我解决了什么问题，创造了什么价值" |
| **场景化延伸** | 基于实际材料，合理延伸想象真实工作场景（访谈、汇报、推动落地） |
| **量化表达** | 用具体数字体现成果（访谈X人、标注X个节点、减少X%沟通成本） |
| **差异化竞争力** | 每个经历都要体现你比别人强在哪里 |
| **直接可用** | 每个输出都要能直接用，不要二次加工 |

### 关键转变：从学生视角到工作视角

❌ **避免学生视角：**
- "我学习了X模块，了解了Y功能，分析了Z问题"
- "我整理了笔记，画了流程图，提出了建议"

✅ **采用工作视角：**
- "客户每天对账上千笔，财务耗时3-4小时（背景）→ 我访谈2位实施同事，拆解规则链发现精确匹配依赖度过高（行动）→ 输出优化方案，导师评价'可纳入Q3规划'（成果）"
- "开发团队要做接口迁移，需要理解复杂业务逻辑（背景）→ 我画了状态流转图，标注12个节点5个异常分支（行动）→ 文档被开发引用，减少30%返工沟通（成果）"

## 📖 使用指南

### 阶段1：经历深挖

**目标**：把工作经历转化为差异化竞争力

**输入**：工作文档文件夹（学习笔记、会议纪要、项目文档等）

**输出结构**：
```markdown
# [公司] - [岗位] - 面试版

## 一、公司背景
## 二、求职（简历 + 自我介绍）
## 三、追问预案（基于关键词）
## 四、素材索引
## 五、面试速查卡
```

**核心问题**：
1. 背景：客户/团队当时面临什么核心问题？
2. 角色：你在这个问题中承担什么责任？
3. 行动：你采取了哪些具体行动？（访谈X人、画X个图、参加X会）
4. 成果：有什么量化数字？（标注X个节点、减少X%沟通）
5. 反馈：你的产出得到了什么反馈？（导师评价、开发引用）

### 阶段2：JD 定制

**目标**：让简历通过筛选 + 判断是否值得投递

**输入**：JD 文本 + 选择的经历

**输出**：
- 公司调研（Markdown）：业务模式、行业地位、匹配度分析
- 简历定制（Markdown）：针对 JD 优化的简历描述

### 阶段3：面试准备

**目标**：面试时任何问题都能答

**输入**：JD + 经历深挖内容 + 公司调研

**输出**：HTML 面试手册（美观易查）

## 🎨 特色功能

### 1. 场景化延伸
基于你的实际工作材料（笔记、会议纪要），合理延伸想象真实工作场景：
- 不是"我学习了收银台模块"
- 而是"客户每天对账上千笔，我访谈2位同事收集痛点"

### 2. 量化表达
所有成果都用具体数字体现：
- 访谈X位同事
- 标注X个状态节点
- 梳理X个异常分支
- 减少X%沟通成本

### 3. 追问预案
基于简历关键词，预测面试官可能问的问题：
- 每个关键词配 1 个主问题 + 2-3 个延伸问题
- 提供结构化回答思路
- 附相关文档链接（Obsidian 双链格式）

### 4. 面试速查卡
面试前5分钟快速回顾：
- 核心数字
- 关键词速记
- 可能被问到的难点及答案

## 🛠️ 技术实现

OfferGet 是一个 **AI Skill**，设计为在支持 Skill 机制的 AI 助手（如 Claude、Claude Code）中运行。

**核心机制**：
- 通过 `SKILL.md` 定义技能入口和流程
- 读取用户本地工作文档
- 基于设计哲学生成结构化输出
- 输出 Markdown/HTML 格式的求职材料

**文件流转**：
```
工作文档 → AI 分析 → 面试版经历 → 简历定制 → 面试手册
   ↑                                              ↓
   └────────────── 持续迭代优化 ←─────────────────┘
```
