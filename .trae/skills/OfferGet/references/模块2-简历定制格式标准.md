# 模块2：简历定制（场景化+量化）格式标准

> **核心原则**：基于经历深挖的场景化内容，针对JD关键词优化表达
> **关键转变**：从"罗列经历" → "匹配JD关键词+量化成果"

---

## 输入标准

**用户提供：**
- JD文本或链接
- 从 `02-我的经历/` 中选择要用的经历（已场景化+量化）

**交互式提问（必须！）：**

```
1. 你想放哪些经历？（从 02-我的经历/ 中选择，按优先级排序）
   - 示例：
     1. 浙江优财云链 - 产品实习生（收银台优化、应付单据梳理）
     2. 深圳惊叹科技 - 产品实习生（付费转化优化）

2. 这段经历中，你最想突出的3个场景是什么？
   （引导用户选择最有量化数字、最有反馈影响的场景）

3. 你的个人评价是什么？（现有版本）
   （用于优化，而非重新生成）

4. 你希望突出哪些能力？（扣住JD关键词）
   （例如：B端产品设计、数据分析、跨部门协作）
```

## 质量标准

1. **关键词匹配**：优化后的表述必须扣住JD关键词（出现3次以上）
2. **量化保留**：经历中的量化数字必须保留（访谈X人、标注X个节点）
3. **场景化保留**：保持"背景-行动-成果"的结构，不变成干巴巴的功能罗列
4. **直接可用**：生成的内容要可以直接复制使用，不需要二次修改
5. **差异化**：突出与JD匹配的独特优势，而非通用能力
