# 模块1：经历深挖（STAR法则）格式标准

## 输入标准

**用户提供：**

- 工作文档

**AI读取：**

- 提取关键工作场景

## 交互式提问标准

**必须问的问题：**

1. 这段工作的背景是什么？（公司/团队/项目情况）
2. 你主要负责什么？具体做了哪些事？
3. 遇到的最大挑战是什么？怎么解决的？
4. 你觉得最有成就感的事是什么？
5. 有什么量化成果吗？（如果没有，AI会帮你延伸）
6. 这段工作让你学到了什么？

## 输出标准

### 文件命名

`[公司名称]-[岗位名称].md`

### 文件结构

```markdown
# 经历深挖报告 - [公司名称] - [岗位名称]

## 一、工作场景提取
从工作文档中识别出以下关键场景：
- 场景1：xxx
- 场景2：xxx
- 场景3：xxx

## 二、STAR完整版（面试讲述用，3-5分钟）

### S - 情境（Situation）
[背景：公司/团队/项目情况，面临的挑战]

### T - 任务（Task）
[目标：需要完成什么任务，解决什么问题]

### A - 行动（Action）
[做法：具体采取了哪些行动，为什么这么做]
1. [行动1 + 原因]
2. [行动2 + 原因]
3. [行动3 + 原因]

### R - 结果（Result）
[成果：量化结果 + 影响 + 学到了什么]
- 数据成果：xxx
- 业务影响：xxx
- 个人成长：xxx

**💡 成果延伸建议（如果用户无法提供）：**
- 合理成果1：xxx（理由：结合公司业务和现实情况）
- 合理成果2：xxx（理由：结合公司业务和现实情况）

## 三、简历精简版（150字以内）
[用"动词+对象+结果"结构，突出关键成果]
示例：负责xxx项目，通过xxx方法，实现xxx成果，提升xxx指标xx%

## 四、面试追问预案
Q1: [可能被问的问题] → [建议回答]
Q2: [可能被问的问题] → [建议回答]
```

## 质量标准

1. **不要假大空**：内容要具体，不要太AI
2. **成果延伸**：结合公司业务和现实情况给出合理建议
3. **量化成果**：尽可能用数字说话
4. **可讲述性**：面试时能流畅讲述3-5分钟
