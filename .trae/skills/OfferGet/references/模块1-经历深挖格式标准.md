# 模块1：经历深挖（场景化+量化）格式标准

> **核心原则**：不是"我做了什么"，而是"我解决了什么问题，创造了什么价值"
> **关键转变**：从"学习-分析-建议"学生视角 → "背景-行动-成果"工作视角

---

## 输入标准

**用户提供：**
- 工作文档文件夹路径（包含学习笔记、会议纪要、项目文档等）

**AI读取：**
- 提取关键工作场景和业务问题
- 识别可量化的工作成果线索
- 寻找导师反馈、团队协作等证据

---

## 交互式提问标准（必须问！）

### 核心6问（基于场景化延伸原则）

```
基于文档内容，我需要了解几个关键问题：

1. **背景**：客户/团队/项目当时面临什么核心问题？
   （例如：客户每天对账上千笔，财务耗时3-4小时；开发迁移项目需要理解复杂业务逻辑）
   ❌ 不要："公司是什么业务"
   ✅ 要："具体什么痛点，影响了谁，有多严重"

2. **角色**：你在这个问题中承担什么责任？
   （例如：分析对账效率痛点、梳理状态机支持开发迁移、快速学习新业务模块）
   ❌ 不要："我是产品实习生"
   ✅ 要："我负责解决XX问题，支持XX项目"

3. **行动**：你采取了哪些具体行动？（要有场景感和量化）
   （例如：
   - 访谈2位实施同事收集客户Top3痛点
   - 画状态流转图，标注12个节点5个异常分支
   - 3天内学习票据池模块，整理20+术语表
   - 参加周会汇报，提出2个架构问题）
   ❌ 避免："我学习了X模块，分析了Y功能"
   ✅ 要："我访谈X人、画X个图、参加X会、整理X个概念"

4. **成果**：有什么量化数字证明你的价值？
   （例如：标注12个状态节点、梳理3万字文档、减少30%返工沟通、提出12项优化建议）
   （如果没有确切数字，基于材料合理延伸：访谈X人、梳理X个场景、输出X页分析）

5. **反馈与影响**：你的产出得到了什么反馈？产生了什么影响？
   （例如：导师评价"思路清晰可纳入Q3规划"、文档被开发团队引用、被邀请参加专项会）
   ❌ 不要："我觉得很有收获"
   ✅ 要："导师/同事/客户具体说了什么，产生了什么实际影响"

6. **差异化**：这件事体现了你什么独特能力？
   （例如：业务翻译能力、结构化思维、快速学习能力、跨部门协作）
```

## 质量标准

1. **场景化**：每个bullet point都要有真实的业务场景（客户痛点、项目需求）
2. **量化**：用具体数字体现工作量和成果（访谈X人、标注X个节点、输出X万字）
3. **反馈**：包含导师/同事/客户的具体反馈和评价
4. **影响**：说明产出产生了什么实际影响（被引用、减少返工、被邀请参会）
5. **可讲述性**：面试时能流畅讲述3-5分钟，有逻辑、有细节、有方法论
